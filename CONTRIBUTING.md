# Contributing to PrayerTimes

Thank you for your interest in contributing to PrayerTimes! We welcome contributions from everyone. Here's how you can help:

## Getting Started

1. **Fork the repository** - Click the fork button on GitHub
2. **Clone your fork** - `git clone https://github.com/YOUR_USERNAME/prayer-times.git`
3. **Create a branch** - `git checkout -b feature/your-feature-name`
4. **Make your changes** - Edit files and test locally
5. **Commit your changes** - `git commit -m "Add your feature description"`
6. **Push to your fork** - `git push origin feature/your-feature-name`
7. **Create a Pull Request** - Go to the original repo and click "New Pull Request"

## Development Setup

### Prerequisites
- Any modern web browser (Chrome, Firefox, Safari, Edge)
- A code editor (VS Code recommended)
- Optional: Live Server VS Code extension for local testing

### Running Locally
1. Open the project folder
2. Use Live Server extension (right-click `index.html` → Open with Live Server)
3. Or use any local server: `python -m http.server 8000`
4. Navigate to `http://localhost:8000`

## Code Standards

### HTML
- Use semantic HTML5 elements
- Keep structure clean and organized
- Add meaningful comments for complex sections

### CSS
- Use Tailwind CSS utility classes
- Follow the existing color scheme (Islamic Green: #059669)
- Keep custom CSS to a minimum

### JavaScript
- Use modern ES6+ syntax
- Keep functions modular and well-documented
- Test cross-browser compatibility
- Add comments for complex logic

## Types of Contributions

### Bug Reports
- Use the Issues tab on GitHub
- Include detailed description and reproduction steps
- Mention your browser and device

### Feature Requests
- Open an issue with the "enhancement" label
- Describe the feature and why it would be useful
- Suggest implementation approach if possible

### Documentation
- Improve README.md
- Add code comments
- Create guides or tutorials
- Fix typos and grammar

### Code
- Fix bugs and issues
- Add new features
- Improve performance
- Optimize code quality

## Testing Your Changes

Before submitting a PR:
1. Test on multiple browsers (Chrome, Firefox, Safari)
2. Test on mobile devices (iOS and Android)
3. Test offline functionality (Service Worker)
4. Check PWA installation works
5. Verify all prayer times calculations

## Pull Request Guidelines

- Provide a clear title and description
- Reference any related issues (#123)
- Keep commits clean and well-organized
- Ensure your code follows the existing style
- Add tests if applicable
- Update documentation if needed

## Questions or Need Help?

- Check existing issues and discussions
- Open a new discussion for questions
- Ask in the issue comments
